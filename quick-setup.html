🔧 הגדרה מהירה

⚠️ חשוב לקרוא!

בטח תהיתת למה הכפתורים בממשק הניהול לא עובדים... הסיבה פשוטה:

המערכת לא יודעת לאיזה פרויקט בגיטהאב לחבר!

צריך להגדיר את זה פעם ראשונה, ואז הכל יעבוד חלק כמו שעון.

1צור פרויקט בגיטהאב (אם עדיין לא עשית)

עבור ל-GitHub.com וצור repository חדש:

  • לחץ על הכפתור הירוק "New repository"
  • תן לו שם כמו uniguide או unistream-guide
  • הגדר אותו כ-Public (אם אתה רוצה שהאתר יהיה זמין לכולם)
  • לחץ "Create repository"

שמור את הכתובת! היא תיראה כך:

https://github.com/USERNAME/REPOSITORY-NAME.git

2הרץ הגדרה אוטומטית

הדרך הכי פשוטה - לחץ לחיצה כפולה על הקובץ:

setup-git.bat

הסקריפט יעשה הכל אוטומטית:

  • יאתחל Git בתיקייה
  • יבקש ממך את כתובת הרפוזיטורי
  • יבקש את השם והדואר שלך
  • יעלה את הקבצים הראשוניים

3חלופה - הגדרה ידנית בטרמינל

אם אתה מעדיף לעשות זה בעצמך, פתח טרמינל בתיקיית הפרויקט והרץ:

git init
git remote add origin https://github.com/USERNAME/REPOSITORY-NAME.git
git config --global user.name "השם שלך"
git config --global user.email "email@example.com"
git add .
git commit -m "הגדרה ראשונית"
git push -u origin main

4בדוק שהכל עובד

אחרי ההגדרה, נסה:

publish.bat

או:

npm run update

אם זה עובד - מזל טוב! המערכת מוכנה לעבודה! 🎉

✅ אחרי ההגדרה הראשונית:

  • הכפתורים בממשק הניהול יעבדו - יציגו לך את הפקודות הנכונות
  • הסקריפטים יעבדו - publish.bat ו-npm run update
  • הפרסום יהיה אוטומטי - שינוי → לחיצה → פרסום לאתר

🤔 למה זה לא עבד בלי הגדרה?

כי המערכת פועלת כך:

  1. הממשק (admin.html) - זה דף HTML שרץ בדפדפן שלך מקומית
  2. הדפדפן לא יכול לגשת לגיטהאב ישירות - זה לא בטוח
  3. לכן הכפתורים מציגים לך פקודות טרמינל - שאתה מריץ בעצמך
  4. הסקריפטים צריכים לדעת לאן לשלוח - לכן ההגדרה הראשונית
