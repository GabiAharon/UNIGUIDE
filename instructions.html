💡 המפתח להבנה:

המערכת עובדת בשני שלבים נפרדים:

  1. עריכה מקומית: אתה עורך תוכן בממשק הניהול, השינויים נשמרים במחשב שלך
  2. פרסום לאתר: כשאתה מרוצה, אתה מפרסם את השינויים לגיטהאב והאתר מתעדכן

🚀 הפעלה ראשונה

1 פתח את ממשק הניהול

פתח את הקובץ admin.html בדפדפן שלך (Chrome, Firefox, Edge)

⚠️ חשוב: פתח את הקובץ ישירות מהתיקייה (file:// protocol), לא דרך שרת אינטרנט.

2 התנסה בעריכת תוכן

נסה להוסיף נושא חדש או לערוך נושא קיים. השינויים נשמרים אוטומטית בדפדפן שלך.

3 פרסם שינויים מקומית

לחץ על "פרסם שינויים מקומית" כדי לעדכן את הקבצים במחשב שלך.

4 צפה בתוצאה

פתח את index.html או quiz.html החדשים בדפדפן וראה את השינויים שלך!

☁️ פרסום לגיטהאב

🎯 בחר את השיטה הנוחה לך:

🖱️ דרך ממשק הניהול

במקטע "גיבוי ושחזור" יש 3 כפתורים:

  • פרסם שינויים מקומית: עדכון הקבצים במחשב
  • פרסם לגיטהאב: העלאה אוטומטית (יציג הוראות)
  • פרסם עם הודעה מותאמת: העלאה עם הודעה שלך

⚡ לחיצה כפולה על קובץ

הכי פשוט - לחץ לחיצה כפולה על:

publish.bat

הקובץ יעשה הכל אוטומטית!

⌨️ דרך טרמינל/PowerShell

פתח טרמינל בתיקיית הפרויקט והרץ:

npm run publish
# או
npm run update
# או
node publish.js

📝 עם הודעה מותאמת אישית

node publish.js "הוספתי נושא חדש על Salesforce"
# או
publish-custom.bat
🔐 הערה חשובה: אם זו הפעם הראשונה שלך, ייתכן שתצטרך להתחבר לגיטהאב או להגדיר SSH keys. הסקריפט יסביר לך מה לעשות אם יש בעיה.

🔧 פתרון בעיות נפוצות

❌ "לא נמצאו שינויים לפרסום"

זה אומר שלא שינית כלום מאז הפרסום האחרון. זה בסדר!

❌ שגיאה בחיבור לגיטהאב

בדוק:

  • האם יש לך חיבור לאינטרנט
  • האם התחברת לחשבון גיטהאב בטרמינל
  • הרץ:
    git config --global user.name "השם שלך"
    git config --global user.email "הדואר שלך"

❌ התמונות לא עולות

וודא:

  • הקובץ קטן מ-5MB
  • הפורמט נתמך (JPG, PNG, WebP, GIF)
  • השם לא מכיל תווים מיוחדים

❌ השינויים לא נשמרים

בדוק:

  • האם הדפדפן מאפשר localStorage
  • נסה לרענן את הדף ולנסות שוב
  • אם בפיירפוקס - ודא שהגדרות הפרטיות מאפשרות שמירה מקומית

🆘 כלי עזרה וכתובות מועילות

  • בדיקה מהירה:
    node test-functions.js --quick
  • איפוס מערכת:
    npm run setup
  • עזרה לסקריפט:
    node publish.js --help
